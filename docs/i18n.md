# Multi-Language Support (i18n)

This application supports both English (EN) and Farsi/Persian (FA) languages.

## How to Use

### Accessing Different Languages

- **English**: Navigate to `/en/` or any path starting with `/en/`
- **Farsi**: Navigate to `/fa/` or any path starting with `/fa/`

The root URL (`/`) automatically redirects to `/en/` by default.

### Switching Languages

Click the globe icon (🌐) in the header to open the language menu and select your preferred language:
- English 🇬🇧
- فارسی 🇮🇷 (Farsi)

The language switcher will maintain your current page path when switching languages.

## Features

- **URL-based routing**: Language is part of the URL path
- **RTL Support**: Farsi pages automatically display with right-to-left layout
- **Automatic locale detection**: Middleware handles locale validation
- **Full translation**: All UI text, menu items, and categories are translated

## For Developers

### Adding New Translations

1. Add your translation keys to both language files:
   - `/messages/en.json` (English)
   - `/messages/fa.json` (Farsi)

2. Use the `useTranslations` hook in components:
   ```tsx
   import { useTranslations } from 'next-intl'
   
   const t = useTranslations()
   
   // Use translations
   <h1>{t('home.welcome')}</h1>
   ```

### Translation File Structure

```json
{
  "app": { ... },
  "home": { ... },
  "menu": { ... },
  "categories": { ... },
  "items": { ... },
  "cart": { ... },
  "reviews": { ... },
  "navigation": { ... }
}
```

### Supported Locales

Currently supported locales are defined in `/i18n.ts`:
- `en` - English
- `fa` - Farsi/Persian

To add more languages, update the `locales` array in `/i18n.ts` and create a corresponding JSON file in `/messages/`.

## Technical Details

- **Framework**: Next.js 14 App Router
- **i18n Library**: next-intl
- **Routing**: Dynamic `[locale]` segment
- **Middleware**: Automatic locale detection and validation
- **CSS**: Custom RTL support in `globals.css`
